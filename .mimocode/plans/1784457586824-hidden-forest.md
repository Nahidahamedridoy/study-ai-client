# Dashboard Redesign Plan — Premium AI SaaS Dashboard

## Context
The current dashboard is functional but basic: a static sidebar, simple stat cards, and a recent resources table. The goal is to redesign the entire dashboard experience into a premium AI SaaS look (Linear/Vercel/Notion AI style) while keeping **all existing APIs, backend endpoints, and data fetching logic unchanged**.

The project already has all needed libraries installed: Framer Motion, Recharts, Lucide React, Tailwind CSS v4, and a custom ThemeProvider with dark mode support.

## Step-by-Step Implementation

### Step 1: Update `globals.css` — Add animation utilities
Add to existing CSS:
- `@keyframes shimmer` for skeleton loading effect
- `.sidebar-collapsed` transition width utility
- `.glass-sidebar` variant for dark glassmorphism sidebar
- Gradient border utility for cards

### Step 2: Update `src/app/layout.js` — Conditional Navbar/Footer
Use `usePathname()` to detect `/dashboard` routes and hide `<Navbar />` and `<AppFooter />` when inside the dashboard. This fixes the current issue where both root navbar AND dashboard sidebar render simultaneously.

### Step 3: Rewrite `DashboardSidebar.jsx` — Premium glassmorphism sidebar
- **Glassmorphism**: `bg-white/70 dark:bg-gray-900/70 backdrop-blur-xl` with subtle border
- **Collapsible**: State toggle to collapse to icon-only (64px) vs expanded (256px), with smooth width transition
- **Active route**: Framer Motion `layoutId="sidebar-active"` animated pill behind active item (like Navbar's existing pattern)
- **Lucide icons**: Keep existing icons, add `Search`, `Bell`, `Sun`, `Moon` for topbar
- **Profile section**: Gradient avatar circle, user name/email, subtle card
- **Logout button**: Styled with rose accent, hover animation
- **Collapse button**: ChevronLeft/ChevronRight toggle at bottom of nav
- **Logo area**: Brand with text-gradient, collapses to just icon when collapsed

### Step 4: Create `DashboardTopbar.jsx` — Premium top bar
- **Welcome message**: "Good morning, {name}" with time-of-day logic
- **Search bar**: Styled input with Search icon, rounded-full, glass background
- **Notifications**: Bell icon with animated dot indicator
- **User avatar**: Gradient circle with user initials + dropdown hint
- **Breadcrumb**: Auto-generated from pathname (e.g., Dashboard > Add Resource)
- **Theme toggle**: Sun/Moon icon using existing ThemeProvider context
- All using framer-motion for hover/press animations

### Step 5: Create `AnimatedCounter.jsx` — Number animation
- Framer Motion `useMotionValue` + `useTransform` + `animate` to count from 0 to target value
- Duration: ~1.5s with easeOut
- Triggered when element enters viewport (framer-motion `whileInView`)

### Step 6: Create `StatCards.jsx` — Animated statistics
- 4 cards: Total Resources, AI Chats (placeholder from stats or 0), Study Plans (placeholder or 0), Saved Resources
- Each card: gradient icon bg, animated counter, subtle hover scale + shadow lift
- Uses existing `fetchDashboardStats` API — maps `totalResources`, `savedCount`, etc.
- Framer Motion staggered fade-up entrance
- Loading state: shimmer skeleton

### Step 7: Create `ActivityCharts.jsx` — Recharts visualizations
Three charts using Recharts (already installed, v3.9.2):
1. **Weekly Activity** — BarChart with gradient bars (mock data derived from resources' createdAt, or use resource count by day-of-week)
2. **Resource Categories** — PieChart/DonutChart with category distribution from user's resources
3. **Learning Progress** — LineChart showing resources added over time

Data: Derived from `fetchMyResources({ page: 1, limit: 50 })` — compute charts client-side from the resource list (category counts, weekly distribution, monthly trend). No new API needed.

### Step 8: Create `RecentActivity.jsx` — Timeline + history
- **Timeline**: Visual vertical timeline with dots + lines showing recent resource uploads
- **AI History section**: Placeholder cards showing "Recent AI Chat" entries (styled but using 0/empty state since no AI API exists yet)
- Uses existing `fetchMyResources` data, sorted by `createdAt`
- Framer Motion staggered entrance

### Step 9: Create `QuickActions.jsx` — Action cards
- 4 cards: Add Resource, AI Chat, Generate Study Plan, Explore Resources
- Each: Icon, title, description, gradient hover border
- Links to existing routes: `/dashboard/add-resource`, `/explore`
- AI Chat & Study Plan: link to `#` (placeholder — no backend for these yet, but UI is ready)
- Framer Motion hover: scale(1.02) + shadow lift + gradient border reveal

### Step 10: Create `LoadingSkeleton.jsx` — Full dashboard skeleton
- Animated shimmer placeholders matching the final layout
- Sidebar skeleton + topbar skeleton + 4 stat card skeletons + chart skeletons
- Uses existing Skeleton component with enhanced shimmer CSS

### Step 11: Rewrite `DashboardShell.jsx` — New layout orchestrator
- Uses `useAuthGuard` (unchanged)
- Renders: Sidebar + Topbar + main content area
- Handles collapsed sidebar state (pass to Sidebar, adjust main margin)
- Includes ThemeProvider integration
- Loading state uses new LoadingSkeleton

### Step 12: Rewrite `src/app/dashboard/page.js` — Compose everything
- Framer Motion page wrapper with `AnimatePresence` for route transitions
- Compose: Topbar → StatCards → Charts (2-col grid) → RecentActivity + QuickActions (2-col grid)
- All data fetching stays identical (same `useQuery` calls, same service functions)
- Staggered entrance animations via framer-motion variants

## Design System

### Colors
- Primary: Indigo (#4f46e5 → #818cf8) — existing, kept
- Accent: Cyan (#06b6d4) — existing, kept  
- Gradient: Indigo → Cyan for accents
- Surface: White (light) / Gray-950 (dark) — existing

### Spacing
- Dashboard padding: `p-6 md:p-8`
- Card padding: `p-6`
- Grid gaps: `gap-5` to `gap-6`
- Section spacing: `space-y-8`

### Shadows
- Cards: `shadow-sm` → `hover:shadow-lg` transition
- Sidebar: `shadow-2xl` on mobile drawer
- Glassmorphism: `backdrop-blur-xl` + semi-transparent bg

### Border Radius
- Cards: `rounded-2xl` (existing)
- Buttons/pills: `rounded-xl` to `rounded-full`
- Avatars: `rounded-full`

### Typography
- Headers: `font-extrabold` (existing pattern)
- Stats: `text-3xl font-extrabold`
- Body: `text-sm` to `text-base`
- Muted: `text-gray-500 dark:text-gray-400`

### Animations
- Page transition: `opacity 0→1, y 20→0` over 0.3s
- Card entrance: staggered `fadeUp` with 0.05s delay per card
- Hover: `scale(1.02)` + `shadow-lg` transition
- Counter: 0 → N over 1.5s easeOut
- Active route pill: `layoutId` shared animation
- Skeleton: shimmer CSS animation
